Echonym

A highly secure, offline-first anonymous social platform engineered for scale.

A full-stack application built around zero-latency UX: a 3-tier frontend (Riverpod → SQLite → Firestore) for instant startup, and an event-driven Node.js/TypeScript backend where every mutation runs through secure, transactional Cloud Functions instead of direct client writes.

  • Zero-latency startup3-tier architecture (Riverpod → SQLite → Firestore). Background FCM sync silently populates the local cache — the app opens to data instantly, no loading spinner.
  • Audio DSP pipelineReal-time voice anonymization via FFmpeg filter chains, paired with a 60fps waveform visualizer rendered through a hand-built CustomPainter — zero image assets, pure canvas math.
  • AI moderation at uploadEvery photo and display name is routed through a Hive AI moderation pipeline before going live, with Firestore security rules enforcing pending/rejected states.
  • Event-driven backendNode.js/TypeScript Cloud Functions handle all mutations via Firestore triggers — zero client-side writes. Hourly cron jobs apply logarithmic decay to trending scores.
  • Discovery engineAlgolia semantic search combined with geospatial bounding-box queries and a multi-variable popularity ranking algorithm — all computed server-side.

Genspire

Where startup builders find their next co-founder — not their next follower.

A full-stack social portfolio and collaboration platform for builders, solo-built from the ground up: a Gemini-powered semantic recommendation engine, Agora RTC voice lounges with a mutex-guarded state machine, and an offline-first sync pipeline. Every layer — auth, security rules, backend Cloud Functions, and real-time presence — designed and shipped solo.

  • Gemini vector recommendation engineCloud Functions auto-vectorize every build and profile via embedding-001. getPersonalizedFeed runs cosine similarity against the user's onboarding intent vector, filtered at a 0.20 threshold — real vector math, not keyword search.
  • Mutex-guarded Agora state machineJoin/leave operations are serialized through a custom mutex in a ~1,500-line service — handling stale-session detection, token renewal 5 minutes before expiry, live connection-quality badges, and ghost-participant cleanup.
  • TOCTOU-proof username claimsUsername changes run inside a single Firestore transaction — read old handle, check new handle availability, atomically swap both documents. No race condition possible, even under concurrent requests.
  • Offline-first repository patternFeed loads from local SQLite in ~50ms on cold start, while a background Dart isolate silently syncs new builds via FCM topic notifications — the UI never blocks on network.
  • Force-update gating + crash reportingEvery cold start checks a Firestore minRequiredVersion doc and blocks the app behind an update dialog if outdated. Crashlytics is wired into Flutter's global error handlers, tagged to the authenticated user.

Ravl

A high-frequency real-time connection engine disguised as a dating app.

Built to handle aggressive concurrent state mutations and rapid UI rebuilds. Features a character-by-character live chat system, idempotent mini-game state machines, and a sophisticated push notification lifecycle manager—all operating under strict 60fps rendering constraints via pure Canvas drawing and micro-optimized Riverpod subscriptions.

  • O(1) Matchmaking & Lock CollapsingThe matching engine uses distributed RTDB locks to prevent concurrent claim race conditions. By collapsing 100+ sequential lock reads into a single tree snapshot, the matchmaking latency was cut by 400-900ms per user.
  • Cost-free Algorithmic WaveformsInstead of extracting and storing heavy audio amplitude arrays in Firestore, voice message waveforms are pseudo-randomly generated on the fly using the audioUrl.hashCode as a stable seed. Both clients see identical waveforms, costing $0 in database storage.
  • High-frequency RTDB WebSocketsLive chat streams character-by-character via Firebase Realtime Database. Uses single atomic .update() calls to prevent state tearing, and onDisconnect() listeners to instantly clear ghost typing states if a client drops cell service.
  • Idempotent Game ResolutionMini-game timeouts (e.g. Rock Paper Scissors) are highly susceptible to race conditions. The backend resolves these via strict Firestore transactions, ensuring if both clients submit a timeout request simultaneously, the first locks the state and the second safely no-ops.
  • Zero-Read Unread BadgingInstead of opening a global Firestore listener for the unread badge, the totalUnreadCountProvider derives its state purely from the already-cached Riverpod memory stream, saving thousands of unnecessary database reads daily.
  • Self-healing Push InfrastructureThe custom notification router parses FCM multicast response arrays in real-time. If Google returns invalid-registration-token, a Cloud Function uses FieldValue.arrayRemove() to instantly prune the dead token, preventing permanent bandwidth leaks.
  • Keystroke Latency OptimizationProfiling revealed that registering RTDB onDisconnect() listeners inside the text-changed loop added a 100-300ms penalty per keystroke. Hoisting this listener strictly to initState restored instant type-ahead performance.

Solo Inspector

AI-assisted report writing for home inspectors.

A focused tool built for the US home-inspection market. It eliminates hours of manual data entry by processing rough field notes and photos into a formal, legal PDF report entirely on-device.

  • Gemini Vision PipelineIntegrates the gemini-flash-latest multimodal model via the firebase_ai SDK to automatically analyze inspection photos and expand rough shorthand notes into professional, legally appropriate observation sentences.
  • Context-Aware AI PromptingThe UI dynamically aggregates form data (e.g., system type, material, condition) from the current wizard step and injects it as silent system context into the Gemini API. This allows the LLM to write highly specific deficiency notes without the inspector needing to re-type background details.
  • Offline-first Riverpod WizardManages a complex, multi-system inspection state (Roof, Electrical, Plumbing, etc.) using Riverpod notifiers. All mutations instantly commit to Firestore's local offline cache, allowing inspectors to work seamlessly in basements without cell service.
  • Immutable State ArchitectureThe entire 10-step inspection wizard operates on a massive immutable data structure. The state notifier handles complex nested mutations (e.g., adding a photo to a specific subsystem) via copyWith, ensuring predictable UI rebuilds and zero state-tearing.
  • On-device Image CompressionAggressively resizes and encodes inspection photos locally using flutter_image_compress before they hit the network. This minimizes memory overhead during PDF generation and significantly reduces payload sizes sent to the Gemini Vision API.
  • On-device PDF GenerationA custom programmatic layout engine built with the pdf package. It dynamically compiles hundreds of data points, photo grids, and vector signature blocks into a multi-page, executive-summary PDF entirely on the client side without relying on backend rendering.
  • Automated Image CaptioningThe custom PhotoCaptureGrid maps each photo's strongly-typed enum to specific structural components (e.g., "Shingles, flashing, gutters"). When a photo is taken, it pushes the compressed image and context to Gemini Vision to automatically generate precise, professional image captions, completely eliminating keyboard typing in the field.
  • Legal Signature Lock-inThe SignaturePadWidget uses raw canvas tracking to capture vector signatures, instantly writing them to local storage as transparent PNGs. It enforces a visual lock-in state in Riverpod to securely hold the signature, preventing accidental erasure before final PDF compilation.

Voice Khata

Speak your ledger entries — built for local shopkeepers.

A voice-assistant-driven ledger app that lets shopkeepers record sales and credit entries by speaking, instead of typing into a register by hand. Engineered with an NLP intent-parsing pipeline and custom audio streaming lifecycle.

  • Urdu Speech-to-Text PipelineIntegrates the Deepgram API using the nova-3 model to transcribe spoken Urdu audio directly into text, highly optimized with custom keyterms for ledger vocabulary like udhaar, vasooli, and khata.
  • AI Intent ExtractionRoutes the raw transcribed text through a Gemini-Flash language model. Using strict system instructions, it acts as an NLP parser to automatically extract the customer name, numerical amount, and transaction type (debit/credit) into a structured JSON payload.
  • State Machine ArchitectureEmploys a custom RecordingNotifier via Riverpod to manage the complex async state machine of the audio lifecycle—seamlessly transitioning between idle, recording, processing, and confirming states.
  • Offline-first SQLite StorageAll parsed transactions are saved to a local SQLite database with a Singleton DatabaseHelper. This guarantees that ledger data is securely stored on-device and instantly accessible without requiring an internet connection for reads.

Horolux

A secure, AI-powered digital vault for luxury watch collectors.

A purpose-built Flutter app that treats a watch collection like a financial asset portfolio. It combines Gemini AI vision scanning, AES-256 encrypted offline storage, an interactive analytics engine, and a cinematic showcase mode — all in a single, offline-first app.

  • Gemini Vision ScannerThe ScannerScreen accepts a live camera capture or gallery image and immediately pipes the raw Uint8List bytes into gemini-flash-latest via the google_generative_ai SDK. The model returns a strict JSON payload containing brand, modelName, and referenceNumber. The app then fires a second Gemini call with an expert-appraiser system prompt to estimate the current USD secondary-market value. A custom AnimationController drives a triple-ring pulsing scanner UI during the two-stage AI pipeline.
  • AES-256 Encrypted VaultThe StorageService orchestrates a fully encrypted Hive box named watch_vault. On first launch it calls Hive.generateSecureKey(), encodes it as a Base64 URL string, and writes it to the device keychain via flutter_secure_storage. Every subsequent launch reads the key, decodes it, and opens the box with HiveAesCipher — meaning the entire collection is AES-256 protected at rest with zero plaintext footprint.
  • Portfolio Analytics EngineA dedicated _AnalyticsData model computed from the live vault state calculates total portfolio value, cost basis, unrealised gain/loss percentage, total service expenditure, and a 5-year maintenance health score. This data drives an interactive fl_chart donut chart (brand value distribution with touch-to-reveal slice percentages) and animated horizontal progress bars ranking every piece by ROI.
  • Provenance Passport PDFThe PdfExportService constructs a multi-page A4 document entirely on-device using the pdf package. It programmatically renders a circular hero image (280 × 280 pt, clipped to an oval), branded headers with copper letterpress styling, an asset-details table, a valuation block with the AI-estimated market value highlighted, and a chronological service ledger grid. The finished PDF is shared natively via Printing.sharePdf.
  • Immutable Riverpod StateAll collection mutations flow through a single VaultNotifier extends Notifier<List<WatchItem>>. The WatchItem model implements a full copyWith pattern, letting the detail screen refresh an AI market valuation in-place without recreating the entire object. Every write calls the Hive box directly and then triggers a synchronous _loadWatches() to re-push state, guaranteeing the UI is always consistent with on-disk data.
  • Cinematic Showcase ModeA full-screen ShowcaseScreen enters SystemUiMode.immersiveSticky and renders the collection via a PageView with a custom HeavyPageScrollPhysics (mass: 120, stiffness: 80) that mimics the weighted feel of turning a heavy catalog page. Each card computes a real-time parallax offset from _currentPageValue - index that pans the photo horizontally as the card enters the viewport, with an optional film-grain noise overlay composited via BlendMode.softLight.
  • Service Ledger & Health AlertsThe WatchItem model embeds a computed isServiceDue getter that checks whether more than 5 years have elapsed since lastServiced ?? purchaseDate. The analytics screen surfaces a CircularProgressIndicator-based collection-health ring, and the details screen injects an amber warning banner for overdue pieces. The ledger supports attaching receipt photos, comma-separated parts lists, and cost logging — all persisted to the encrypted Hive box via copyWith.
  • Glassmorphism UI SystemA reusable GlassContainer widget wraps every card surface with a BackdropFilter (sigma 15) to produce a live-blurred frosted glass effect. It applies a directional LinearGradient (top-left → bottom-right) at 15% / 5% white opacity in dark mode, bordered with a 1.5 pt translucent stroke — building a cohesive premium UI language that extends from the analytics dashboard to the detail and ledger screens.
